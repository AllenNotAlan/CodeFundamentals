## 🏗️ What .NET Roles Expect

**Technical Bar:**
- Deep C# knowledge (generics, async/await, memory model, LINQ)
- ASP.NET Core internals (middleware pipeline, DI, configuration)
- Entity Framework Core (queries, migrations, performance pitfalls)
- Cloud-native patterns on Azure or AWS (.NET on containers, serverless)
- Testing strategies (unit, integration, architecture tests)

**Common .NET Interview Topics:**
- Value types vs reference types, boxing/unboxing
- `async`/`await` — task scheduling, `ConfigureAwait`, deadlocks
- Dependency injection lifetimes (Transient, Scoped, Singleton)
- Middleware pipeline and request lifecycle
- `IEnumerable<T>` vs `IQueryable<T>` — deferred execution pitfalls
- Garbage collection, `IDisposable`, `Span<T>`, memory pressure

**Interview Format (typical):**
1. **Phone screen** — 30–45 min C# coding or .NET knowledge
2. **Live coding** — 60–90 min practical problem in C#
3. **System design** — 45–60 min architecture with .NET stack
4. **Behavioural** — 45 min values & collaboration
5. **Deep dive** — .NET internals, performance, or domain-specific

## 🗺️ How to Use This Guide

1. **Start with Coding** — .NET roles often gate on C# fluency
2. **System Design** is critical for senior roles — know the Azure/AWS equivalents
3. **API Design** overlaps heavily with day-to-day .NET work
4. Work through each doc's questions, attempt answers before reading solutions
5. Practice writing C# without IDE assistance — interviewers notice reliance on IntelliSense
